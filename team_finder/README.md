# Team Finder

Веб-приложение для поиска участников в проекты с системой тегов навыков и фильтрацией по ним. Учебный проект Яндекс Практикума, **вариант 3** (навыки проектов и фильтрация).

Демо-аналог: <http://team-finder.tech>

## Стек

- **Python 3.11** / **Django 5**
- **PostgreSQL 16**
- **Docker Compose**
- Django Templates + ванильный JS (без фронт-фреймворков)

## Структура

```
team_finder/
├── config/                 # Настройки Django (settings/urls/wsgi/asgi)
├── users/                  # Кастомная модель User (email-login), регистрация, профиль
├── projects/               # Проекты, CRUD, join/finish, management-команда сидера
├── skills/                 # Навыки и AJAX-эндпойнты (autocomplete/add/remove)
├── templates/              # HTML-шаблоны
├── static/                 # CSS + JS
├── docker-compose.yml
├── Dockerfile
├── entrypoint.sh           # миграции → collectstatic → сидер → runserver
├── requirements.txt
├── .env.example
└── manage.py
```

## Запуск (для ревьюера)

```bash
git clone <repo-url> team-finder
cd team-finder

# .env уже включён в репозиторий со значениями для локального запуска.
# Если хочется чистый — скопируй .env.example в .env:
# cp .env.example .env

docker compose up --build
```

После сборки откроется по адресу:

- Приложение: <http://localhost:8000>
- Админка: <http://localhost:8000/admin/>

На первом запуске `entrypoint.sh` сам выполнит:

1. `python manage.py migrate` — применит миграции
2. `python manage.py collectstatic` — соберёт статику
3. `python manage.py load_demo_data` — создаст тестовых пользователей, навыки и проекты (идемпотентно: при повторных запусках ничего не делает)

## Что реализовано (чек-лист задания)

### Базовая функциональность

- Главная страница со списком проектов, карточки (автор + аватар, название, описание, дата), пагинация по 12, сортировка по `-created_at`
- Регистрация (имя, фамилия, email, пароль), редирект на логин
- Аутентификация по email + logout
- Страница пользователя с проектами и «Поделиться профилем»
- Редактирование профиля (имя, фамилия, аватар, био, телефон, GitHub)
- Страница проекта: описание, статус, автор, кнопка «Присоединиться», у владельца — «Редактировать» / «Завершить», список участников
- Создание и редактирование проекта
- Список пользователей (12 на странице, сортировка по `-date_joined`), кнопка «Посмотреть контакты»
- Смена пароля для авторизованных
- Админка для всех моделей

### Вариант 3 — навыки и фильтрация

- Блок «Необходимые навыки» на странице проекта
- Владелец может добавлять и удалять навыки **без перезагрузки страницы** (AJAX)
- Автодополнение при наборе; можно создать новый навык прямо из формы
- На `/project/list/` есть панель фильтра по навыкам
- Фильтрация `?skill=<slug>` оставляет только проекты с этим навыком
- Активный фильтр подсвечен, есть кнопка «× Сбросить фильтр»

### Права доступа

- **Гость**: смотрит проекты и профили, не может создавать или присоединяться
- **Авторизованный**: создаёт свои проекты, управляет ими, откликается на чужие, редактирует свой профиль
- **Админ**: полный доступ через `/admin/`

## Полезные команды

```bash
# Логи
docker compose logs -f web
docker compose logs -f db

# Открыть Django shell в контейнере
docker compose exec web python manage.py shell

# Создать ещё одного суперпользователя
docker compose exec web python manage.py createsuperuser

# Полный сброс БД (удалит volume)
docker compose down -v
docker compose up --build
```

## Данные

- БД хранится в Docker volume `postgres_data`
- Загружаемые медиа (аватары) — в volume `media_data`
- Собранная статика — в volume `static_data`

Данные **сохраняются между перезапусками контейнеров** (`docker compose restart`, `docker compose down && up`). Удаляются только через `docker compose down -v`.

## Качество кода

- PEP 8 (max line length 100)
- Логически разделённые приложения: `users`, `projects`, `skills`
- AJAX-эндпойнты возвращают JSON и проверяют права (`is_owner`)
- CSRF включён везде
